Device Permissions that App Uses

  • Usage Access Permission

    Material Lock needs to use Usage access permission to detect which application you opened and lock it.

  • Coarse Location Access

    Used to get Wi-Fi SSID for automatically enabling/disabling lock service.

  • Device Administrator

    We are getting android.permission.BIND_DEVICE_ADMIN permission for only prevent app uninstall. When someone try to uninstall app, he must remove Device Administrator in device settings.

  • Use Credentials & Manage Accounts

    Used to auto fill Remote Lock feature email field and resetting pin code by checking Google Account credentials.
